│                         📊 REPORTS GENERATED                                │
└─────────────────────────────────────────────────────────────────────────────┘

When you run Get-ElevatedGroupUsage.ps1, these files are created:

1. ElevatedGroupUsage-AllGroups-{timestamp}.csv
   → Complete analysis of all groups
   
2. ElevatedGroupUsage-Top10LeastUsed-{timestamp}.csv
   → Groups with least activity (cleanup candidates)
   
3. ElevatedGroupUsage-Top10MostUsed-{timestamp}.csv
   → Groups with most activity (monitoring priorities)
   
4. ElevatedGroupUsage-EmptyGroups-{timestamp}.csv
   → Groups with no members (if any found)
   
5. ElevatedGroupUsage-CriticalRisk-{timestamp}.csv
   → Groups with 180+ days inactivity (if any found)
   
6. ElevatedGroupUsage-Summary-{timestamp}.csv
   → Summary statistics
   
7. ElevatedGroupUsage-Report-{timestamp}.html
   → Interactive visual dashboard

┌─────────────────────────────────────────────────────────────────────────────┐
│                     💡 KEY CONCEPTS                                         │
└─────────────────────────────────────────────────────────────────────────────┘

Usage Score (0-100%):
  • 80-100% = Highly active (monitor closely)
  • 50-79%  = Moderately active (regular reviews)
  • 25-49%  = Low activity (validate business need)
  • 0-24%   = Minimal/no activity (consider removal)

Risk Levels:
  • Critical = 180+ days inactive (immediate review)
  • High     = 90+ days inactive (validate need)
  • Medium   = 30+ days inactive (watch trend)
  • Low      = Active within 30 days (normal)

Activity Tracking:
  • Based on lastLogonTimestamp AD attribute
  • ~14 day replication delay (acceptable)
  • Considers all user members
  • Excludes disabled accounts

┌─────────────────────────────────────────────────────────────────────────────┐
│                    ⚙️ PARAMETERS REFERENCE                                  │
└─────────────────────────────────────────────────────────────────────────────┘

Get-ElevatedGroupUsage.ps1 supports:

  -OutputFolder <path>
    Where to save reports (default: current directory)
    
  -IncludeNestedMembers
    Recursively enumerate nested group members
    
  -Top <int>
    Number of top/bottom groups to report (default: 10)
    
  -DaysInactive <int>
    Days to consider a group inactive (default: 90)
    
  -IncludeAllGroups
    Analyze all security groups, not just elevated

Examples:
  .\Get-ElevatedGroupUsage.ps1
  .\Get-ElevatedGroupUsage.ps1 -DaysInactive 60 -Top 20
  .\Get-ElevatedGroupUsage.ps1 -IncludeAllGroups -IncludeNestedMembers
